介绍​

Electron-Builder 官网
Electron-Builder 中文网

本次我们使用Electron-Builder
它除了deb和rpm包，还可以构建其他一些兼容的包，比如AppImage、Snap

  • 据说Snap运行速度慢，包很臃肿，权限问题不好解决 （根据我在 Armbian 上的体验，确实如此）
  • Electron-Forge也可以构建Snap。但我试了，失败了
  • 其实还不止这些，但是apk会出 bug
  • 也许可以有Arch Linux的软件包.pkg.tar.zst，Nix，国产Deepin的玲珑uab，等我有空再研究

搭建环境​

强烈推荐用GitPod

Arch Linux​

sudo pacman -Sy --needed nodejs npm git fakeroot dpkg rpm libxcrypt-compat flatpak flatpak-builder elfutils fuse

只能在Arch Linux上这么安装Node.js和npm

Ubuntu​

sudo apt update
sudo apt install git fakeroot snapd dpkg flatpak flatpak-builder elfutils rpm fuse

npm​

npm i electron electron-builder --save-dev

yarn​

yarn add --dev electron electron-builder

平台​

  • Flatpak类似于Snap
  • apk是安卓Android的软件包格式 可以正常构建，但是无法安装，GitHub 上有人提过Issues了，一直没人解决，官方估计也不管了，反正我也有出过apk 的教程，期待Electron-Forge吧

Flatpak​

如果不换源，下载真的非常慢
上海交通大学镜像
但是换了有的时候有会有奇怪的问题，所以还是推荐Gitpod或者Github Codespace

# 安装开发依赖
sudo flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo
sudo flatpak install org.freedesktop.Platform/x86_64/20.08
sudo flatpak install org.freedesktop.Sdk/x86_64/20.08
sudo flatpak install org.electronjs.Electron2.BaseApp/x86_64/20.08

如果报错，要获取更多信息可以这么做：

npm i -g electron-builder
# 要先全局安装Electron-Builder才能执行下面这行命令
env DEBUG="@malept/flatpak-bundler" electron-builder build --linux flatpak

Flatpak 官方的文档，和这个略有不同

运行 AppImage​

  1. 打开终端
  2. 运行./应用名称.AppImage
  3. 如果提示权限不够，就运行chmod +x 应用名称.AppImage
  4. 缺什么包就装什么，顺便跟我说一声

图标​

官方的文档非常难看懂

  • Mac 至少要 512x512 像素
  • Windows 至少要 256x256 像素

兼容性​

  • Node.js和Bun.js都可以正常构建
  • 可以在x86架构上构建ARM架构的包
  • 支持的芯片架构：x64 ia32 armv7l arm64
  • 用法：加上芯片架构参数，如：electron-builder --linux deb --arm64
  • 如果你想测试一下ARM的程序能否运行，可以使用qemu来测试。网上有教程，我懒得折腾，没有具体研究
  • 不能跨操作系统构建！！！
  • 最后支持32位的版本是v18.3.15
